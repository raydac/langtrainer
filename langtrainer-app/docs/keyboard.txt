Virtual keyboard requirements

1. Because it is possible that keyboard of user doesn't have some chars then we need add way to show virtual keyboard in separated moveable and resizable windeo
2. Visual configuration of keys should repeat standard Keyboard of PC but without extended keys. Only chard and numbers also chard ' ', ',', '.', '(', ')', '-'
3. AbstractLangTrainerModule should return enum with languages allowed by a module, by default ENG, it is possible have several keyboards to be used in module so they can be switched by special key with abbreviate of current keyboard language
4. Required ENG (English), RUS (Russian) and EST (Estonian) keyboards
5. Button to activate virtual keyboard shown near close button for module with special icon
6. As only keyboard shown the activating icon is hidden
7. Keyboard can be hidden by special button on form of keyboard and in the case keyboard is hidden and icon to actiate it again shown
8. Clicks on the virtual keyboard buttons should be send into AbstractLangTrainerModule#onCharClick(char symbol)
9. Clicks on real keyboard also should be sent to active module through AbstractLangTrainerModule#onCharClick(char symbol)
10. Virtual keyboard has SHIFT button to switch capitalize chars if toggled. Chars should reflect its state.
