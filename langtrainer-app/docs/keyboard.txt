Virtual keyboard requirements
--------------------------------

1. Users may lack physical keys for some characters; the app provides a separate movable virtual keyboard window.
2. Layout mimics a PC keyboard (letters, digits, and punctuation row including space, comma, period, parentheses, hyphen) per selected KeyboardLanguage layout.
3. When the virtual keyboard is first opened for a session, LangTrainerApplication passes the active module’s AbstractLangTrainerModule#getSupportedLanguages() into VirtualKeyboardWindow — only those layouts appear in the keyboard. The base AbstractLangTrainerModule implementation returns English only; Dialog, Fly game, Crossword, and Editor return KeyboardLanguage.VIRTUAL_BOARD_ALL (ENG, SPA, GER, EST, FIN, LIT, LAT, RUS, UKR, HEB, CZE, ESP, SWE, NOR, DAN, DUT — see enum KeyboardLanguage for the exact row definitions).
4. When several layouts are available, the user switches layout from the keyboard UI (abbreviation / language control on the keyboard window).
5. A non-focusable toolbar button next to the module close control opens the virtual keyboard (icon from resources) without taking focus from the active module field.
6. While the virtual keyboard window is visible, the toolbar trigger may be hidden until the keyboard is closed again (see LangTrainerApplication / VirtualKeyboardWindow behaviour).
7. The virtual keyboard can be dismissed from its own UI; the toolbar button returns when the window is closed.
8. Clicks on virtual keys first insert into the focused editable Swing text component; otherwise they invoke AbstractLangTrainerModule#onCharClick(char symbol) on the active module.
9. Typed characters from the physical keyboard are also forwarded to the active module via onCharClick when the app-wide dispatcher is active (see LangTrainerApplication).
10. The virtual keyboard provides a SHIFT toggle; displayed key caps reflect the current shift state.
11. The virtual keyboard trigger, window, and all keyboard controls are mouse-only and non-focusable; the app restores the previous module focus after opening the keyboard, so the active field, table cell, or module keeps keyboard focus while users click virtual keys, change layout, shift, or hide the keyboard.
