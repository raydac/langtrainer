Shared resource index — classpath location
--------------------------------------------

File: src/main/resources/common/jsons/index.json

This file is the central catalogue of bundled JSON language resources (dialog / word-list /
crossword payloads in the format described in JSON_format.txt). The application loads it at
runtime, loads each referenced JSON file, and builds the module resource picker tree from each
file's optional "path" field (see JSON_format.txt). Folders are shown as collapsible groups in the
UI (single click toggles open/closed).


Root object
-----------

The root must be a JSON object with exactly one top-level array:

  "resources": [ ... ]

Each element of "resources" is a LEAF entry pointing to one JSON file on the classpath. Order in
the array is preserved when placing root-level resources; folder groups appear when the first
resource with a given path prefix is seen.


Leaf entry (resource)
---------------------

A leaf points to one JSON file on the classpath.

Required fields:

  "resource"  — string, classpath path to the dialog JSON file.

Allowed form today (enforced by the loader):

  /common/jsons/<file-name>.json

  where <file-name> has no slashes (a single file name under common/jsons). Examples:

    /common/jsons/en-ru-small.json
    /common/jsons/en-ru-crossword-small.json

Example:

  { "resource": "/common/jsons/en-ru-small.json" }


Menu path and modules (in each resource JSON, not in index.json)
----------------------------------------------------

Grouping for the picker comes from the optional "path" string on each dialog JSON file (see
JSON_format.txt):

  - If "path" is missing or blank, the resource appears at the list root.
  - Path segments are separated only by slash (/) or backslash (\)
    (for example "travel/russian-english/50 words"). Spaces and other characters may appear inside
    a segment; each segment is trimmed when parsed.
  - Segment matching is case-insensitive; labels in the UI show each segment with only the first
    character capitalized (e.g. "english" and "eNGLISH" both display as "English").

Example — nested groups via path on the JSON file:

  { "path": "By size/50 terms", "menuName": "...", ... }

Module restriction comes from the optional "modules" array on each resource JSON file (see
JSON_format.txt):

  - If "modules" is missing or empty, all modules may list the resource.
  - Plain ids are allow rules: if any plain id is present, only listed modules may list the resource.
  - Ids prefixed with "!" are deny rules: the named module must not list the resource.

Module id values (enum LangTrainerModuleId, must match exactly, optional "!" prefix for deny rules):
DIALOG, FLY_GAME, CROSSWORD, BRICKS, BARS, IMAGES.

Example:

  { "modules": [ "DIALOG", "FLY_GAME" ], "menuName": "...", ... }
  { "modules": [ "!IMAGES" ], "menuName": "...", ... }


Filtering and visibility
------------------------

1) Classpath gate
   Only paths matching the allowed pattern under /common/jsons/ are loaded. Invalid or missing
   paths fail fast when the index is read.

2) Module gate ("modules" on the resource JSON)
   If "modules" is absent or empty on the loaded resource, all modules may list it.
   Plain module ids restrict the resource to listed modules. "!MODULE" entries exclude that module
   from the picker even when no plain ids are present.

3) Empty branches
   A folder whose subtree contains no leaf visible to the current module is not shown for that
   module.

Implementation reference: ClasspathLangResourceIndex, ClasspathResourceIndexTree, ResourceMenuPath,
LangTrainerResourceAccess.visibleToModule, AbstractLangTrainerModule.isResourceAllowed.


Validation errors
-----------------

The loader throws IllegalStateException with a clear message if:

  - the root is not an object or has no "resources" array;
  - an index entry still uses the removed folder shape ("name" + "children") or "modules";
  - a leaf is missing "resource" or the path is not under /common/jsons/.


UI behaviour (brief)
--------------------

- List order follows index order for root items; within a folder, resources appear in index order.
- Folders start collapsed; single-click a folder row to expand or collapse.
- Double-click / Start applies only to resource (leaf) rows, not folder headers.

For the JSON schema of each pointed-to resource file, see JSON_format.txt in this folder.

For which application modules consume these entries, see application_description.txt (registered modules) and the Module_*.txt notes.
