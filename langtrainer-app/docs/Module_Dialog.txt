Module DIALOG
---------------

1. It is a child of AbstractLangTrainerModule.
2. The main idea: show text in one language and ask the user to type the translation (or the paired line), with fuzzy checking so minor punctuation / case / near-miss characters can still count as correct.
3. GUI layout of the main work area (two columns):

+-------------------------------------------------------------+
|                           |                                 |
|                           |                                 |
|                           |                                 |
|                           |                                 |
|                           |                                 |
|    Zone show A            |  Zone show B                    |
|                           |                                 |
|                           |                                 |
|                           |                                 |
|                           |                                 |
|                           |                                 |
+---------------------------+---------------------------------|
|  Zone enter A             |  Zone enter B                   |
|                           |                                 |
+-------------------------------------------------------------+

4. Dialog content is loaded from JSON in the DialogDefinition shape (see JSON_format.txt).
5. Bundled classpath lists come from common/jsons/index.json (see index_json.txt): optional folder hierarchy, collapsible rows in the picker, then a leaf JSON file under /common/jsons/. The user may also "Open from file" to load any compatible JSON from disk (shown in the list with a leading * in the menu title).
6. Before play, the module shows the resource list; after choosing a resource, the user picks which side (language A or B) they will type; the other side is the prompt target.
7. The non-active entry zone shows the expected line in the target language; the user types in the chosen language zone.
8. On Enter, entered text is compared to the expected line (TypingComparisonUtils — not strict equals; punctuation and similar characters may be tolerated). On success, both show zones advance; the exercise continues line by line.
9. Optional UI: shuffle order, show phrase, typing tip — see module toolbar and toggles in code.
10. When all lines are completed, a full-window modal banner shows "Hurraa! Completed!" (see DialogModule). Clicking the banner (or waiting 10 seconds) dismisses the overlay and returns to the resource selection card inside the module (not the same as exiting the whole application).
11. Small sample bundled resources exist under /common/jsons/ and are wired through index.json.
