Implementation requirements
------
1. Java 17 based project.
2. Maven is the build system (langtrainer-app/pom.xml configures exec-maven-plugin with mainClass LangTrainer).
3. Swing for GUI; image resources may be PNG, JPG, or SVG (SVG via Batik in ImageResourceLoader).
4. Prefer minimal external dependencies; SVG rendering uses a dedicated library.
5. Application starts from the command line; Maven exec plugin is configured for a standard launch.
6. On start the application shows a splash screen for 5 seconds while the main window loads; the splash does not block loading.
7. Base Java package for most code: com.igormaznitsa.langtrainer
8. Package layout (actual sources under src/main/java):

   LangTrainer.java
      Entry point (default package). Invokes LangTrainerApplication on the Swing EDT.

   com.igormaznitsa.langtrainer
      Modules.java — enum listing all built-in modules (factories for AbstractLangTrainerModule).

   com.igormaznitsa.langtrainer.api
      AbstractLangTrainerModule, KeyboardLanguage, LangTrainerModuleId, etc.

   com.igormaznitsa.langtrainer.engine
      LangTrainerApplication, main menu, classpath index, virtual keyboard wiring, JSON parsing helpers, etc.

   com.igormaznitsa.langtrainer.modules
      DialogModule, FlyGameModule, CrosswordModule, EditorModule — one package per module area.

   com.igormaznitsa.langtrainer.ui
      Shared UI helpers (fonts, banners, etc.).

   com.igormaznitsa.langtrainer.text
      Text comparison helpers used by typing modules.


Functional requirements
------
1. Language trainer desktop application.
2. Main menu lists registered modules (icon + name). Activation: mouse click, or keyboard (arrow keys + Enter or Space). See MainMenuPanel.
3. Each module is a mini-application extending AbstractLangTrainerModule (name, description, image, control form). A red close control in the module toolbar returns to the main menu.
4. AbstractLangTrainerModule#onActivation is called when a module becomes active.
5. AbstractLangTrainerModule#onClose is called when leaving the module for the main menu.

Registered modules (Modules enum)
------
  DIALOG     — phrase typing / translation practice (Module_Dialog.txt).
  FLY_GAME   — timed word entry with aircraft metaphor (Module_FlyGame.txt).
  CROSSWORD  — generated crossword from word lists (Module_Crossword.txt).
  EDITOR     — JSON resource editor for the same dialog format (Module_Editor.txt).


Java / code style (project norms)
------
1. Prefer DRY and readable structure; use final where values must not change.
2. Comments where non-obvious invariants or behaviour matter; avoid noise.
3. Meaningful method and type names.
4. Streams and records are used where they clarify batch processing or data shapes.


Design
--------
Polished but pragmatic UI. Optional WAV playback is supported (SoundClip; e.g. Fly game SFX). Image loading helpers live under engine (ImageResourceLoader).
