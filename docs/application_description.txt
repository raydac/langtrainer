Implementation requirements
------
1. It is a Java 17 based project
2. Maven is in use as build system for the project
3. Swing as GUI tool to show interface and interact with user, image resources can be provided as PNG, JPG and SVG
4. Single module command line based project, use as less external libraries as possible it is good to use only external library to render SVG
5. Start must be from CLI and also maven exec plugin should have config for start
6. On start the application shows splash image for 5 seconds (it doesn't lock load of application)
7. Common application package com.igormaznitsa.langtrainer
8. Expected package tree

   com.igormaznitsa.langtrainer
      |
      +--api (contains all abstract and api modules)
      +--engine (contains all stuff related to internal work of application not related to modules work)
      +--modules (here added every custom module for application)
      +--Main.java (main class to start application)
      +--Modules.java (enum to have links to all modules of application which added as enum records)

Functional requirements
------
1. Application is a language trainer application
2. Main menu is extendable one and shows list of registered modules as some image and name. Module activated through mouse click by user. Also keyboard navigation must be allowed through cursor keys and enter or space key to activate.
   +-------------------------+
   | /-----\  /-----\        |
   | |Item1|  |Item2|        |
   | \-----/  \-----/        |
   |                         |
   |                         |
   +-------------------------+
3. Every main menu module describes some internal mini application and has ancestor AbstractLangTrainerModule which should provide methods to get description, name, image and internal module control form which will be shown. In right top corner of module should be shown big close button shown as red X and if user click it then it return system to main menu.
4. During activation called AbstractLangTrainerModule#onActivation
5. During exit to the main menu called AbstractLangTrainerModule#onClose for active module


Java requirement
------
1. Use DRY and CLEAN CODE approach
2. Use final modifier if variable or argument should not be changed
3. Use single return point for functions
4. Write comments only in places where it must have
5. Write appropriate but not too long names of methods
6. Strongly prefer use of Java Stream API style for code write!

Design
--------
Design must be nice but not very hard, it is possible to add sounds into application so implement some auxiliary player which will be able take wav file and play
Also provide auxiliary classes to load images from resources.
