## 2. Architecture & Technical Discoveries
- External Estonian resource includes `DIALOG`, `FLY_GAME`, `CROSSWORD`, and `BRICKS`, mixing single-word pronouns with short multi-word phrases so different modules have usable rows.
- `InputEquivalenceSupport` uses positional matching when `key` and `value` lists have equal sizes, and any-key-to-any-value matching otherwise.
- Equal-size rows where `key` and `value` contain the same token set now act as equivalence classes, allowing compact reciprocal substitutions in JSON.
- Maven must be run as `mvn22`.
- Assembly descriptors must use `**/*` include patterns so root-level launcher files are packaged.
- GitHub folder sync uses Hub4j `org.kohsuke:github-api` anonymously through the GitHub Contents API.
- `langtrainer-app` declares Apache Commons Lang directly because production code uses `SystemUtils`; Commons Text already pulls the same 3.20.0 version transitively.
- App JSON resource parsing and writing uses Jackson databind 2.21.0; `DialogLine` maps uppercase JSON fields with `@JsonProperty("A")` and `@JsonProperty("B")`.
- `ExternalResourceSupport.syncAndLoadAsync` reports sync failures through `JOptionPane` and Java Util Logging; local external resources remain visible if loading succeeds after a failed sync.
- `GitHubFolderSynchronizer` rejects non-portable GitHub paths before local writes: traversal segments, separators, control characters, Windows-reserved names/chars, trailing dots/spaces, and case-insensitive collisions.
- External resources merge entries from `<externals>/common/jsons/index.json` and `<externals>/index.json`; `<externals>` defaults to an OS-specific app config path (`%APPDATA%\LangTrainer\externals`, `~/Library/Application Support/LangTrainer/externals`, or `$XDG_CONFIG_HOME/langtrainer/externals`) and can be overridden by Java property or environment variable `LANGTRAINER_EXTERNALS`.
- Resource entries in `index.json` may be absolute (`/common/jsons/file.json`) or relative to the `index.json` file location (`./folder/file.json`).
